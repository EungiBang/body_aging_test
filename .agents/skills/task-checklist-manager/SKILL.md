# 목표 및 상태 기반 작업 관리 스킬 (Task Checklist & State Manager)

이 스킬은 복잡하거나 다단계가 필요한 작업을 수행할 때, 임의로 작업을 시작하여 코드를 훼손하거나 방향을 잃지 않도록 돕기 위한 행동 지침입니다.

## 핵심 수행 규칙

1. **작업 시작 전 작업 명세서(task.md) 작성**
   작업을 진행하기 전에 반드시 아래 세 가지 요소를 포함하여 작업 파일(`.agents/task.md` 또는 아티팩트 `task.md`)을 작성하거나 갱신해야 합니다.
   - **Plan (계획)**: 무엇을 왜 수정하는지 목적과 대상을 명확히 정의합니다.
   - **Checklist (체크리스트)**: 단계별로 검증 가능한 체크박스 목록을 만듭니다.
     - `[ ]` 미완료 작업
     - `[/]` 진행 중인 작업 (현재 집중하고 있는 단일 항목)
     - `[x]` 완료된 작업
   - **Context Notes (맥락 노트)**: 수정 시 가정한 사항, 중요 아키텍처 결정 정보, 제약 조건 등을 기록합니다.

2. **단일 작업 루프 및 독립적 검증**
   체크리스트의 각 항목을 진행할 때마다 진행 중 상태(`[/]`)로 마킹하고, 완료 시(`[x]`)로 마킹하며 실시간으로 `task.md` 파일을 업데이트하여 에이전트 간 연속성을 보장합니다.

3. **종료 조건 및 검증 절차 준수**
   모든 체크리스트가 완료되면 최종 빌드나 컴파일 테스트를 거쳐 정상 동작을 확인한 후, 수행된 요약 결과를 기록한 walkthrough 보고서를 제출해야 합니다.
