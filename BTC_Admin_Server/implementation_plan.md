# IBEL 명상지도 사례보고서 자동화 시스템 개발 계획서

본 계획서는 원장님(관리자) 전용 백엔드 기능인 **[IBEL 사례보고서 생성 및 출력]** 기능의 구현 방향을 정리한 문서입니다. AI 측정 결과를 기반으로 사실(Fact) 중심의 보고서를 자동 생성하고, 이를 A4 양식으로 출력할 수 있도록 설계합니다.

## 🎯 기능 목표
1. **관리자 전용 기능**: 일반 회원은 접근할 수 없고, 원장님(Admin)만 회원 관리(HistoryManager)에서 접근하여 작성 및 조회 가능하도록 구현.
2. **환각(Hallucination) 방지**: AI가 소설을 쓰지 않도록 측정 결과(수치, 상태)에 기반한 팩트 위주의 진단 요약을 생성.
3. **IBEL 템플릿 완벽 매핑**: 원장님이 제공해주신 문서 양식(1~6번 항목)을 웹 UI 및 인쇄물(A4) 양식으로 완벽히 구현.
4. **출력(Print) 최적화**: 작성 완료 후 종이 문서로 보관하거나 제출할 수 있도록 브라우저 인쇄(`@media print`) 기능 최적화.

---

## 📋 워크플로우 및 요구사항 정의

1. **대상 선택 및 입력 (1, 2번)**
   - 관리자가 회원 기록(`HistoryManager`)에서 특정 회원의 **수련 전(Before) 기록**을 선택하여 보고서 작성을 시작합니다.
   - 원장님이 수동으로 **1번(프로파일)**과 **2번(호소문&니즈)**을 작성합니다.
2. **AI 자동 생성 및 수정 (3, 4, 5번)**
   - 수련 전(처음) 측정한 데이터 팩트를 기반으로 AI가 **3번(진단 요약), 4번(원인 분석), 5번(개입 전략)**의 초안을 자동 작성합니다.
   - 생성된 3, 4, 5번 항목은 원장님이 자유롭게 **텍스트를 직접 수정(에디팅)**할 수 있습니다.
3. **변화 결과 작성 (6번)**
   - 수련 후 재측정한 **After 기록**을 선택하여 불러오면, Before와 After의 데이터 상태를 비교할 수 있도록 세팅되며, 이를 바탕으로 6번 항목을 완성합니다.
4. **출력물 디자인 (A4 워드 양식)**
   - 제공해주신 워드(.docx) 파일의 표(Table) 형식을 웹 화면에 그대로 구현합니다.
   - 브라우저 인쇄 시, 일반 워드 문서를 출력한 것과 동일한 깔끔한 A4 폼으로 출력되도록 CSS(`@media print`)를 최적화합니다.

## 💻 제안하는 변경 사항

### 1. 타입 및 인터페이스 확장
#### [MODIFY] `BTC_Admin_Server/types.ts`
- `CaseReport` 인터페이스 신규 추가:
  - `clientProfile`: 내담자 핵심 프로파일 (수동 입력)
  - `complaint`: 호소문 및 니즈 (수동 입력)
  - `diagnosisSummary`: 진단 요약 (AI 팩트 기반 초안 -> 수동 수정 가능)
  - `causeAnalysis`: 핵심 원인 분석 (AI 팩트 기반 초안 -> 수동 수정 가능)
  - `interventionStrategy`: 개입 전략 (AI 팩트 기반 초안 -> 수동 수정 가능)

## ✅ 검증 계획

1. **데이터 기반 팩트 체크**: 임의의 측정 기록으로 AI 자동 완성을 실행하여, 실제로 측정되지 않은 질병이나 증상을 지어내지 않는지 철저히 테스트.
2. **A4 인쇄 레이아웃 테스트**: 브라우저 인쇄 기능(Ctrl+P) 실행 시, 표지나 페이지가 잘리지 않고 워드 양식과 동일하게 1장~2장에 깔끔하게 들어가는지 확인.
3. **권한 제어**: Lite 버전이나 일반 사용자 화면에서는 절대 이 기능이 노출되지 않도록, 관리자 서버(Admin Server) 프로젝트 내부에만 캡슐화.
