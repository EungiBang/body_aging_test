# BTC 3바디 AI분석기 기술 개발서 (Technical Specification)

이 문서는 'BTC 3바디 AI분석기'의 핵심 동작 원리, 최적화 기법, 그리고 향후 v2.0 및 그 이후 버전에서 반드시 유지되어야 할 핵심 기술 명세(Technical Spec)를 기록합니다.

## 1. 버전 1.6 핵심 업데이트 및 최적화 내역 (v2.0 필수 이관 항목)

### 1.1 하드웨어 메모리 누수 방지 (WebGL 최적화)
- **이슈**: TensorFlow.js의 Pose Detection 모델이 프레임당 생성하는 WebGL 텍스처가 브라우저 VRAM에 누적되어, 장시간 켜둘 시 앱이 느려지거나 크래시(블루스크린) 발생.
- **해결 공식**:
  ```typescript
  // 텍스처 즉시 비우기 강제
  try {
      tf.env().set('WEBGL_DELETE_TEXTURE_THRESHOLD', 0);
  } catch(e) {}
  
  // 프레임 단위 스코프 관리
  tf.engine().startScope();
  const poses = await globalDetector.estimatePoses(offCanvas);
  tf.engine().endScope();
  ```
- **v2.0 적용 지침**: 뇌 테스트(Brain Test) 모듈 복구 시, 해당 모듈 내부의 `estimatePoses` 호출부에도 **동일한 스코프 관리 및 텍스처 즉시 비우기 로직이 반드시 적용되어야 함**.

### 1.2 실시간 균형 테스트 트래킹 (Cheat Prevention)
- **이슈**: 균형 테스트(한발 서기)에서 "15초간 흔들리다가 사진 찰칵하는 마지막 순간에만 동작을 취하는 꼼수"를 클라우드 AI(제미나이)가 완벽한 자세로 오판하는 치명적 맹점 존재.
- **해결 로직 (`usePoseEstimation.ts` 기준)**:
  1. **Foot Drop (발 디딤) 감지**:
     - 매 프레임 양쪽 발목(`left_ankle`, `right_ankle`)의 Y좌표 차이 계산.
     - 양 발목의 Y좌표 차이가 어깨너비의 40% 미만으로 좁혀지면, "공중에 뜬 발을 내려놓았다"고 판별.
     - 디딤 횟수가 1회를 초과할 시 강제 감점 플래그 생성.
  2. **Sway (상체 흔들림/요동침) 감지**:
     - `nose`(코) 좌표의 프레임 간 X 이동 거리 누적.
     - 어깨 너비 대비 누적 이동 픽셀을 정규화하여 `swayScore` 산출.
     - `swayScore`가 50을 초과하면 심각한 흔들림으로 간주.
- **클라우드 연동 (`geminiService.ts`)**:
  - 마지막 스냅샷 이미지와 함께 `balanceData` 메타데이터(`footDrops`, `swayScore`)를 프롬프트에 동봉.
  - "사진 속 자세가 정상이더라도, 현장에서 측정한 footDrops가 1회 이상이면 점수를 대폭 깎고 코멘트에 꼼수임을 명시할 것"이라는 강제 지시어를 통해 AI가 눈속임에 속지 않도록 강력 통제.
- **v2.0 적용 지침**: 
  - 향후 유연성 테스트나 기타 다이나믹한 동작 검사(예: 팔 들어올리기 유지 등)에서도 이와 같은 로컬 판별(Cheat Prevention) 로직 + 메타데이터 전송 구조를 재사용해야 함. 사진 1장이라는 태생적 한계를 극복하는 핵심 설계 구조임.

### 1.3 스쿼트 자동 카운팅 정확도 향상 (Side-View 최적화)
- **이슈**: 스쿼트를 정면(Front-View)에서 진행할 경우 관절(골반, 무릎, 발목)이 카메라 기준 Y축 상에서 앞뒤로 겹치는 오클루전(Occlusion) 현상 발생. 2차원 카메라 AI가 하체 굴절 각도를 정확히 연산하지 못해 자동 카운팅이 실패하는 구조적 결함.
- **해결 로직 (`usePoseEstimation.ts` 및 `CameraModule.tsx`)**:
  1. **가시성 기반 추적 타겟 스위칭**:
     - 피사체가 우측/좌측 어느 쪽으로 측면을 향하든 대응할 수 있도록, 양쪽 다리의 신뢰도 점수(Confidence Score: `hip.score + knee.score + ankle.score`)를 합산.
     - 더 높은 점수를 가진 다리를 추적 기준 다리로 자동 판별.
  2. **측면 최적화 각도(Angle) 튜닝**:
     - 정면일 때의 모호한 각도를 버리고, 측면 기준 다운(Down)은 `angle < 110`, 업(Up)은 `angle > 150`으로 정밀 세팅.
  3. **측면 스쿼트 전용 UI 도입**:
     - 밋밋한 사각형 가이드 대신, 수직선(Plumb Line), 엉덩이 타겟 박스, 무릎 한계선 등 측면 동작에 최적화된 가이드라인 렌더링.
- **v2.0 적용 지침**:
  - 스쿼트, 런지 등 하체 관절이 전후방으로 겹치는 모든 측정 모듈은 반드시 **측면(Side-View)** 전용 가이드라인을 제공하고 좌/우측 자동 탐지 로직을 사용하는 것으로 표준화함.

### 1.4 푸시업 자동 카운팅 유연성 향상 (Diagonal 최적화 및 기준 완화)
- **이슈**: 푸시업을 정면으로 측정 시 팔꿈치가 가려지고, 측면으로 측정 시 반대편 팔과 몸이 합쳐지는 현상 발생. 더욱이 기존의 카운트 기준이 엘리트 체육인 수준(내려갈 때 90도 미만)으로 세팅되어 일반 회원의 카운트 불만 야기.
- **해결 로직 (`usePoseEstimation.ts` 및 `CameraModule.tsx`)**:
  1. **대각선 45도 트래킹**:
     - 피사체가 대각선으로 엎드릴 경우, 가장 화면에 뚜렷한 팔(Confidence 점수 합산이 높은 팔)을 메인 센싱 타겟으로 동적 전환.
  2. **일반인 범주로 각도 판정 완화**:
     - 기존의 깐깐한 직각 룰을 폐기하고, **다운(Down) 시 `angle < 120`**, **업(Up) 시 `angle > 140`**으로 대폭 완화하여 일반인/노약자의 현실적인 가동 범위를 수용.
  3. **대각선 전용 구도 가이드라인(UI) 렌더링**:
     - 45도 대각선 형태의 3D 공간을 표현하는 면(Plane)과 가시적 타겟 박스를 추가하여 "대각선으로 엎드려야 함"을 직관적으로 유도.
- **v2.0 적용 지침**:
  - 맨몸 운동 등 상체 전체 굴절이 일어나는 동작은 **대각선 45도 구도**를 원칙으로 하며, 모든 카운팅 각도는 일반 사용자의 체력(실무 현장)을 기준으로 완화된 룰을 적용할 것.

## 2. 보안 체계 구조
- **지점망 통제 (Branch Auth)**:
  - `machineId` 기반 라이선스 검증 방식을 사용. 관리자 대시보드(Firebase)에서 데이터를 삭제할 경우, 클라이언트의 오프라인 딜레이(Grace Period)를 거쳐 앱 사용이 영구 차단.
  - v2.0 업그레이드 배포 시, 하위 버전 사용을 강제 차단할 수 있도록 auth 구조 확장이 필요함.

## 3. 버전 3.1: 절대 평가 점수제 및 가중치 신체 나이 환산 아키텍처

### 3.1. 기존 상대적 연령 추정 방식의 폐기 사유
- **이슈**: 기존 시스템은 각 측정 항목(스쿼트, 푸시업, 균형 등)별로 AI가 즉각적인 '신체 나이(예: 32세, 45세)'를 자체 산출하도록 프롬프트에 의존함. 이로 인해 AI의 할루시네이션(환각) 및 '착한 AI 편향'이 발생하여 심각한 체력 미달자에게도 실제 나이보다 10살 이상 젊게 판정해주는 **점수 인플레이션 현상**이 지속됨.

### 3.2. 100점 만점 엄격 채점(Hard-coded Scoring) 로직 도입
- **해결 로직 (`geminiService.ts` 기준)**:
  1. **근력 항목 (스쿼트, 푸시업)**:
     - `getSquatAvgReps`, `getPushupAvgReps` 함수를 통해 성별/연령대별 평균 횟수를 도출.
     - **평균 달성 시 70점** 부여.
     - **평균의 150%(1.5배) 달성 시 100점** 부여 (상위 10% 기준).
     - 0회 이하 시 0점 처리 후 선형 보간(`70 + ((validReps - avg) / (top10 - avg)) * 30`).
  2. **균형 항목 (눈 감고 한발 서기)**:
     - 클라이언트 단에서 수집된 `footDrops`(발 닿음 횟수)와 `swayScore`(흔들림 수치)를 2D 매트릭스 공식으로 직접 100점 환산.
     - 0회 닿음 & 흔들림 < 20 : 100점
     - 1회 닿음 & 흔들림 < 20 : 70점
     - 2회 이상 닿음 : 40점 이하 폭락 (충격 요법)
  3. **자세/유연성 항목 (AI 프롬프트 제어)**:
     - 프롬프트에 `[절대 원칙: 100점은 완벽한 국가대표급 상위 1%에게만 부여하고, 일반인 평균은 60~70점 범위로 평가하라]`는 강제 제약 주입.

### 3.3. 가중치 기반 최종 신체 나이(Physical Age) 역산 공식
- **가중치 합산 (`overallScore`)**:
  - `스쿼트(20%) + 푸시업(15%) + 균형(25%) + 자세(20%) + 유연성(10%) + 팔올리기(10%)`의 비율로 100점 만점의 종합 점수 산출.
- **나이 환산 (Age Reverse-Calculation)**:
  - 종합 점수 **70점을 실제 나이(0 증감)**의 기준점으로 설정.
  - **1점당 0.4살 증감** (예: 100점 만점 시 `-12살`, 50점 이하 시 급격한 노화 판정).
  - 최대 -12살 / 최대 +20살 방어선(Hard Cap) 구축.

- **v4.0 적용 지침**: 
  - 향후 평가 항목이 추가되거나 센서 데이터가 도입되더라도, **모든 개별 항목은 반드시 100점 만점 기준의 절대 점수로 1차 환산**되어야 하며, AI에게 직접 나이를 추정하게 하는 로직은 절대 사용해서는 안 됨.
